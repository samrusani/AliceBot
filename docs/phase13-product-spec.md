# Phase 13 Product Spec

## Phase Theme
Alice Lite + Integration Ergonomics

## Objective
Turn Alice from a powerful continuity system into an easier-to-adopt, easier-to-feel, easier-to-integrate product for solo users, builders, and agent teams.

## Phase Principle
Make Alice easier to adopt without weakening its semantics.

## Why This Phase Exists
Phase 12 improved Alice's quality substrate:
- hybrid retrieval
- explicit mutation
- contradiction and trust handling
- public evals
- task-adaptive briefing

Phase 13 operationalizes that quality for adoption:
- `P13-S1` reduced integration complexity
- `P13-S2` reduced startup and onboarding friction
- `P13-S3` made memory quality and conversation risk visible

## In Scope
- one-call continuity adoption
- Alice Lite deployment profile
- memory hygiene visibility
- conversation health visibility
- simpler first-run and demo value

## Out Of Scope
- retrieval research
- graph migration
- new channels
- new provider/runtime substrate work
- marketplace work
- enterprise/admin expansion

## Success Criteria
- external agents integrate through a simple continuity path
- solo builders can start Alice more easily
- memory hygiene is inspectable
- stale/risky conversations are visible
- the product feels simpler and more legible without becoming semantically weaker

## Final Status
Shipped in `v0.4.0`.
