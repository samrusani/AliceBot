# Phase 13 Sprint Plan

## Sequence
- `P13-S1` One-Call Continuity
- `P13-S2` Alice Lite
- `P13-S3` Memory Hygiene + Conversation Health

This order is intentionally non-redundant:
- `P13-S1` reduces integration complexity
- `P13-S2` improves first-run experience
- `P13-S3` makes memory quality more visible

Create the default continuity integration surface across API, CLI, and MCP.

### Status

## P13-S2: Alice Lite

### Objective

### Status

## P13-S3: Memory Hygiene + Conversation Health

### Objective
Make duplicates, stale facts, unresolved contradictions, review pressure, stale threads, risky threads, and thread health visible and actionable.

### Scope
- duplicates
- stale facts
- unresolved contradictions
- weakly trusted memory
- review queue pressure
- recent threads
- stale threads
- risky threads
- thread health posture

### Acceptance Criteria
- hygiene issues are visible and actionable
- stale or risky conversations are visible without deep system knowledge
- the sprint improves operational trust without becoming new substrate work

### Status
Shipped
